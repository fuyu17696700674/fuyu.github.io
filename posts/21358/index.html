常用的命令

hexo server

hexo s
启动服务器。默认情况下，访问网址为： http://localhost:4000/

预览的同时可以修改文章内容或主题代码，保存后刷新页面即可；
对hexo的config.yml修改，需要重启服务器

hexo deploy

hex deploy
自动生成网站静态文件，并部署到设定的仓库

如果设置了github会上传到github上

hexo clean

hexo clean
清除缓存文件 (db.json) 和已生成的静态文件 (public)。

在网站出现异常的时候试试

hexo new

hexo n 文章名称
在source/posts中新建一篇md文件

新建文章可以直接在post中删除，再使用hexo g重新生成静态
hexo new page x 是在source中生成目录
文章的标题是可以再md中修改的，所以可以写的简单点

hexo generate

hexo g
生成网站静态文件到默认设置的 public 文件夹。

会重新生成public，之前我之前把照片放到public的img目录就中招了，图片全没了
建议在部署网站前使用

hexo version

hexo version

常用组合

hexo clean && hexo s
hexo clean && hexo d

可以设置快捷
当然我觉得都差不多

文章的属性

  • title 标题 文章的文件名
  • date 建立日期 文件建立日期
  • updata 更新日期 文件更新日期
  • comments 是否开启评论 true
  • tags 标签
  • categgories 分类
  • permalink url的名字
  • cover 封面设置
  • top_img 文章顶部图片

我踩的一些坑

  • public文件是临时生成的，使用hexo clean就会清除掉，所以文件要放到source中
  • 先使用hexo g生成public文件，再使用hexo d上传到github，才能在网上看到
  • 编写md文件，与属性之间一定要有空格
